## 2. Detection gap

**When the system knew:** never, in any signal visible from outside the process. The
mechanical hook exists ‚Äî `internal/stream/relay.go`'s `writeEvent` calls
`rc.SetWriteDeadline` before every write ‚Äî but no metric, log line, or counter records
whether that deadline fired, failed to fire, or was silently unsupported. The code's own
error handling explicitly tolerates `http.ErrNotSupported`
(`if err != nil && !errors.Is(err, http.ErrNotSupported) { return err }`), which means "the
deadline is set but genuinely never triggers" and "the deadline call is silently a no-op for
this server configuration" are indistinguishable from outside infergate.

**When the operator (this campaign) knew:** only after deliberately constructing a raw,
zero-read TCP client and stalling it past the configured deadline ‚Äî a fault-injection test, not
routine operation. No alert fired, because none exists for this condition: there is no
"stream held open past its write-deadline budget" signal anywhere in the observability stack
(Contract 2's canonical metric set does not name one, and none was found in
`inferops/dashboards/golden-dashboard.json`).

**Alert that existed / fired / was missing:** missing entirely. A real slow/stalled client in
production would consume a dispatch slot and an upstream generation slot for as long as it
kept the connection nominally open (bounded here only by the mock engine's own 3000-token
output cap, not by any gateway-side timer), with no operator-facing signal distinguishing it
from a normal long-running stream.

## 3. Root cause

Owning repo: **infergate**. The write-deadline mechanism in
`internal/stream/relay.go` is only partially implemented ‚Äî the file's own header comment
states this directly: *"Write deadline: each write carries `writeTimeout` when the underlying
`ResponseWriter` supports deadlines; a violation fails the write so the caller can release the
upstream. (**Full slow-client fault handling ‚Äî scenario 4 ‚Äî is later work; the bound exists
now.**)"* This campaign's finding does not contradict that comment ‚Äî it is a real,
reproducible confirmation, at the deployed-binary level, that the documented gap is genuine
and observable, not merely theoretical.

**Root-cause hypothesis (not proven from outside infergate; this campaign's job was to verify
deployed behavior against contract, not to instrument infergate internals):** the write
deadline may never actually reach a blocked write in this deployment's configuration, or
`http.ResponseController.SetWriteDeadline` may be silently returning `http.ErrNotSupported`
for the exact `ResponseWriter` implementation in play ‚Äî a hypothesis recorded honestly as a
hypothesis, not a confirmed diagnosis, because confirming it requires instrumentation inside
infergate, out of scope for a black-box fault-injection campaign run from a consumer repo.

## 4. Mitigation

**No mitigation was applied ‚Äî this is a fault-injection finding, not a live incident.** No
client was affected in production; the "stalled" client in Part A was this campaign's own
synthetic test harness. The recorded mitigation path is the action-item list below, routed to
infergate as the owning repo per the program's repo-ownership rule (this repo/inferops does
not patch infergate's source).

**What recovery evidence shows:** once the synthetic client resumed reading, the stream
delivered its remaining content normally and settled as a clean success ‚Äî there is no
lingering resource leak visible in this evidence (`inference_requests_in_flight` returned to
0 in both Part A and Part B). **What was explicitly NOT measured:** gateway memory growth
under a sustained population of stalled slow clients (the abort condition's other clause) ‚Äî
out of this campaign's instrumentation scope, recorded as a coverage gap rather than assumed
passing (`inferops/faults/scenario-04/checklist.md`, `verdict.md`). Normal-speed clients'
TTFT was not measured against an isolated no-slow-population baseline in the same run (21/21
fast-population requests completed normally, but a rigorous head-of-line-blocking bound was
not separately isolated) ‚Äî also recorded as a scope reduction, not asserted as a pass.
